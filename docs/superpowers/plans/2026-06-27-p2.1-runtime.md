# P2.1 — Make Agents Run (Plan)

> REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Checkbox steps.

**Goal:** the "make it actually run" piece. An authored *scheduled* agent fires on its cron; an *action-only* agent fires its bound tools for real (e.g. posts via Postiz); a *missed-call* agent gets a real number; and the editor/deploy surfaces "connect this tool" so bound tools become live. All **money-safe**: live sends/posts fire only when the workspace's tool is actually connected, else the agent records intent + warns.

**Architecture:** additive. A new schedule cron enumerates deployments whose `blueprint.trigger.kind==="schedule"` and fires `runEventAgent` when due. The action-only branch (P2 T6) gains a real tool-invoke via the existing MCP tool-merge runtime. The deploy provisioner gets a missed-call exception (inbound-ish event agents get a number). The editor surfaces unconnected bound tools.

**Conventions:** verify from inside `packages/crm`: `npx tsc` (gate 0), `node --test`, `check-use-server.sh src`, `pnpm build` at phase end. Commit per task, push at end. Work in `icp3-wedge`.
