# Vendoring Magic UI components onto an existing motion engine, comprehension-first

## The problem, in one line
Add "world-class, net-positive" animations across a marketing site without adding a dependency, without shipping decoration, and without a way to gate motion quality (static screenshots can't see movement).

## The approach
1. **Vendor, don't depend.** Magic UI components are MIT copy-paste built on the `motion`/`framer-motion` package. If that package already ships, `WebFetch` each component's canonical doc/registry page (the doc URL is the spec, like a design mockup) and adapt it — never `npm i` a component library. Verify no `package.json` change and no library import in review.
2. **Impose one Adaptation Contract on every vendored component:** `"use client"`; a `useReducedMotion()` early-return that renders a COMPLETE static state (all content at final positions, no motion); a `forceStatic?: boolean` prop that forces that same static branch on demand; colors default to theme tokens/`currentColor` not brand hex; `className` passthrough. The `forceStatic` prop is what later lets a review gallery show the reduced-motion state on demand.
3. **The static state IS the design.** Write each component's test against the static branch (renderToString asserts the real text/children are present) — that proves SSR/crawler-safety and the reduced-motion fallback in one test.
4. **The kill rule makes it net-positive.** Before applying any animation, name the one concept/step/state it clarifies; if the honest answer is "looks cool," cut it. Map each kept component to an existing section AND an architecture claim (e.g. beam = "source of truth pushes outward"; orbit = "any agent on any surface"; terminal = the real install command). This cut 6 of 13 candidate components from the first surface.
5. **Build a dev-only `/motion-lab` gallery as the human gate.** One gated route (404 in prod) rendering every component with its labelled comprehension purpose + a reduced-motion toggle (drives every `forceStatic`) + a light/dark toggle. Motion taste is a human decision; this is the single surface a human reviews. Its honesty is load-bearing: any reused component lacking an internal reduced-motion guard must be gated to the toggle in the lab too, or the gallery lies about production.
6. **Vendored CSS animations must be self-contained.** A Magic UI component that relies on a Tailwind `animate-<custom>` utility (e.g. `animate-orbit`) is INERT if that keyframe isn't in the project's Tailwind config — the animation silently no-ops. Co-locate the `@keyframes` + drive via inline `animation:` so the component works standalone. (Tailwind v4 ships none of Magic UI's custom keyframes.)

## Judgment calls
- **Did NOT wire a full motion-token system.** A `--motion-fast/base/slow` token layer is only honest if something consumes it. `motion`/framer components legitimately use second-based JS durations (the framer idiom) and guard reduced-motion via `useReducedMotion`, so they do NOT read CSS timing tokens. Only CSS-transition-based components (hover) consume the tokens. Shipping tokens that nothing reads — plus a rubric doc asserting "nothing hardcodes ms / components read the tokens" — is a never-lies violation baked into the foundation that every later slice inherits. Fix: wire at least one real CSS-transition consumer + import the token CSS on the route, and write the doc to the TRUTH (two mechanisms: CSS-transition reads tokens; JS motion uses its own durations, reduced-motion-guarded).
- **Did NOT force dark-mode theming on light-only sections.** These marketing sections live in a build-stack that only ever renders light (the dark record-mode stack is separate components). So augments match the section's existing hardcoded-light-hex convention rather than converting to tokens — converting would be redesign/scope-creep. But VENDORED components stay token-themed (they're reused elsewhere incl. the lab's dark toggle).
- **Did NOT keep a pointless hover.** A bento card's hover-translate exists to open room for a CTA; on CTA-less cards it slides content into empty space — a net-NEGATIVE animation. `forceStatic` drops it: the bento's comprehension value is the "one-system" LAYOUT, not the hover. Cutting motion is a valid outcome of the kill rule.
- **Deferred all visual-taste items to the human gate**, fixed only correctness/a11y/honesty in code (empty-link alt text, aria-hidden on decorative duplicates, the token-honesty reconciliation, the lab's reduced-motion faithfulness).

## The reusable rule, one line
A design-token layer (or any shared contract) is only allowed to exist if something consumes it and the docs describe what's actually true — a defined-but-unwired token plus a doc that claims it's honored is a never-lies violation, worst when it's the foundation a later swarm inherits.
